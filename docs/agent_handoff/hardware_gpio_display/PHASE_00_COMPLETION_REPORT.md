### 4.2 Existing incorrect assumptions (in the pre-existing board_pins.h files)

These stem from the older target map and are invalidated by the confirmed
target map in the PHASE 00/01 prompts:

1. `node_sensor_enviroment/include/board_pins.h` assumes MQ-7 AO targets
   GPIO8 and therefore declares a GPIO8 conflict with BME680 SDA
   (`BOARD_TARGET_MQ7_AO_GPIO`, `#warning`, `board_pins_report_conflicts()`).
   New target puts MQ-7 AO on GPIO1 — **the conflict no longer exists**; the
   whole conflict block is obsolete.
2. Same file assumes GP2Y analog targets GPIO35 ("not ADC-capable on S3").
   New target is GPIO2 — the GPIO35 fallback logic is obsolete.
3. Same file assumes an MQ-7 heater GPIO exists. New target: no heater pin.
4. **Live wiring bug in the current uncommitted state**: `BOARD_BME680_SDA_GPIO`
   and `BOARD_MQ7_HEATER_GPIO` are BOTH `GPIO_NUM_8`, and `main.c` initializes
   both. The header comment claims the I2C bus was "held at known-good pins",
   but the value was actually changed to 8, double-driving GPIO8 (I2C SDA vs.
   heater output). It compiles (only the TARGET-macro `#warning` fires, which
   is about a different, obsolete conflict) but is electrically wrong. Removing
   the heater GPIO in PHASE 01 resolves it.
5. `node_rfid_finger_print/include/board_pins.h` assumes RC522 RST targets
   GPIO9 and warns about an OLED-SCL/RST GPIO9 conflict. New target: RST→GPIO14,
   OLED SCL→GPIO9 — **that conflict no longer exists**; the `#warning` block
   and `BOARD_TARGET_RC522_RST_GPIO 9` are obsolete.
